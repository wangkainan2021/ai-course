# 快速启动指南

## 问题：Failed to fetch

这个错误表示前端无法连接到后端服务。

## 立即解决

### 步骤1：启动后端服务

**方法A：使用命令提示符**
1. 按 `Win + R`
2. 输入 `cmd`，按回车
3. 输入以下命令（一行一行执行）：
   ```cmd
   cd /d D:\课程设计\backend
   node server.js
   ```

**方法B：使用PowerShell**
1. 按 `Win + X`，选择 "Windows PowerShell"
2. 输入以下命令：
   ```powershell
   cd D:\课程设计\backend
   node server.js
   ```

**方法C：双击运行**
- 双击 `启动后端.bat` 文件

### 步骤2：确认后端已启动

看到以下信息说明成功：
```
后端服务器运行在 http://localhost:3000
API文档: http://localhost:3000/api
等待请求...
```

**重要**：保持这个窗口打开，不要关闭！

### 步骤3：测试连接

打开浏览器，访问：
- http://localhost:3000/api/courses

如果看到 `{"success":true,"data":[]}` 或类似JSON数据，说明后端正常运行。

### 步骤4：使用前端

现在可以：
1. 打开 `frontend/admin.html`
2. 尝试上传关卡或创建课程
3. 应该可以正常工作了

## 如果仍然失败

### 检查1：端口是否被占用

如果3000端口被占用，修改端口：

1. 编辑 `backend/server.js`，找到：
   ```javascript
   const PORT = 3000;
   ```
   改为：
   ```javascript
   const PORT = 3001;
   ```

2. 编辑 `frontend/admin.html` 和 `frontend/index.html`，找到：
   ```javascript
   const API_BASE_URL = 'http://localhost:3000/api';
   ```
   改为：
   ```javascript
   const API_BASE_URL = 'http://localhost:3001/api';
   ```

3. 重新启动后端服务

### 检查2：防火墙设置

确保Windows防火墙没有阻止Node.js访问网络。

### 检查3：查看详细错误

1. 打开浏览器控制台（F12）
2. 切换到 Network（网络）标签
3. 尝试上传
4. 查看失败的请求，点击查看详细信息

## 一键启动脚本

我已经创建了 `启动后端.bat` 文件，双击即可启动。
