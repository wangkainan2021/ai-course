# 测试说明

## 后端服务状态

后端服务器应该已经在后台运行。如果无法访问，请手动启动：

```bash
cd backend
npm start
```

服务器运行在：http://localhost:3000

## 测试步骤

### 1. 测试后端API

打开浏览器，访问以下URL测试API：

- 获取课程列表：http://localhost:3000/api/courses
- 获取关卡列表：http://localhost:3000/api/levels

如果看到JSON数据，说明后端正常运行。

### 2. 测试管理端

1. 打开 `frontend/admin.html` 文件（在浏览器中打开）
2. 测试上传关卡：
   - 切换到"上传关卡"标签
   - 选择"图片关卡"
   - 填写标题和描述
   - 上传几张图片（可以使用 `images` 文件夹中的图片）
   - 填写图片说明文字（每行一个）
   - 点击"上传图片关卡"
3. 测试创建课程：
   - 切换到"创建课程"标签
   - 填写课程名称和描述
   - 选择刚才上传的关卡
   - 点击"创建课程"

### 3. 测试学生端

1. 打开 `frontend/index.html` 文件（在浏览器中打开）
2. 应该能看到：
   - 顶部有数字导航（1, 2, 3...）
   - 内容区域显示第一个关卡的内容
3. 点击不同的数字，应该能切换不同的关卡

## 常见问题

### 问题1：前端显示"加载课程失败"

**原因**：后端服务没有运行

**解决方法**：
1. 检查后端服务是否运行
2. 打开浏览器控制台（F12）查看错误信息
3. 确认API地址是否正确（默认：http://localhost:3000/api）

### 问题2：无法上传文件

**原因**：文件大小限制或格式不支持

**解决方法**：
- 图片：支持 JPG, PNG, GIF 等常见格式
- 视频：支持 MP4, WebM 等格式
- Canvas：支持 .js 或 .html 文件

### 问题3：Canvas代码不执行

**原因**：代码格式错误或浏览器安全限制

**解决方法**：
- 确保代码格式正确
- 检查浏览器控制台是否有错误
- 代码中应该使用 `canvas` 变量（已自动提供）
